### Slide 3 — 传统方案的局限
- **左列：** 写死的 SQL / 硬编码业务逻辑
  - 灵活性差，新需求需改代码
- **右列：** 纯 Prompt 工程
  - 不稳定，AI 可能"猜错"业务概念
- **共同问题：** 缺乏结构化的业务语义层

---

### Slide 4 — 本体论是什么
- **一句话定义：** 给业务概念建立结构化的"地图"
- **三个核心元素：**
  - Object（对象）— 业务实体，如"股票"
  - Property（属性）— 实体的字段，如"市盈率"
  - Relationship（关系）— 实体间的关联，如"股票 → 财务指标"
- **类比：** 数据库描述"怎么存数据"，本体论描述"数据是什么意思"

---

### Slide 5 — 一个具体例子
- **图示：** `Stock` → `FinancialIndicator` → `ValuationMetric`
- **字段标注示例：**
  - `roe`：百分比类型，净资产收益率
  - `pe_ttm`：倍数类型，滚动市盈率
  - `industry`：分类类型，所属行业
- **重点：** 语义类型让 AI 知道如何理解和格式化每个字段，而不只是知道它是个数字

---

### Slide 6 — Omaha 架构概览
- **流程图：**
  ```
  YAML 配置 → OmahaService → 数据源（模拟/真实）→ 语义格式化 → 用户 / AI
  ```
- **强调两点：**
  1. 配置驱动——不写代码，只改 YAML 就能定义新业务对象
  2. 语义层——结果自动格式化，ROE 显示为 `15.3%`，市值显示为 `¥1.2亿`

---

### Slide 7 — YAML 配置示例
- **代码片段（精简）：**
  ```yaml
  objects:
    - name: Stock
      description: A股基本信息
      properties:
        - name: roe
          type: float
          semantic_type: percentage
          description: 净资产收益率
        - name: industry
          type: string
          semantic_type: category
          description: 所属行业
  ```
- **旁注：** 一个对象定义 = 字段 + 语义类型 + 描述，AI 读到这份配置就知道"ROE 是百分比、代表净资产收益率"

---

### Slide 8 — 演示：自然语言 → 查询结果
- **输入（Chat 界面截图/录屏）：**
  > "找出ROE大于15%的股票，按PE从低到高排列，取前10名"
- **展示：** AI 自动调用 `screen_stocks` 工具，参数自动填写
- **输出：** 格式化结果表格

  | 股票 | 行业 | ROE | PE |
  |------|------|-----|----|
  | 示例A | 银行 | 18.5% | 5.2x |
  | 示例B | 保险 | 16.3% | 7.1x |

- **重点：** 格式（%、x）由语义层自动处理，不是 AI 猜的

---

### Slide 9 — 背后发生了什么
- **四步流程：**
  1. AI 读取本体定义，知道 ROE/PE 的语义
  2. 将自然语言转换为结构化查询参数
  3. OmahaService 执行查询，返回原始数据
  4. 语义层格式化，返回可读结果
- **核心结论：** AI 没有"猜"——它有结构化的知识作为依据，这就是本体论的价值

---

### Slide 10 — 可迁移性
- **标题：** 不只是金融
- **三列并排：**

  | 领域 | 对象示例 | 查询示例 |
  |------|---------|---------|
  | 医疗 | 药品 → 适应症 → 副作用 | "有哪些治疗高血压且副作用少的药" |
  | 供应链 | 商品 → 供应商 → 库存 | "库存低于安全线的核心商品" |
  | 学术 | 论文 → 作者 → 引用 | "某方向被引最多的论文" |

- **核心：** 换一份 YAML 配置，就是一个新领域的 AI 知识库

---

### Slide 11 — MCP 集成（技术亮点）
- **一句话：** Omaha 遵循 MCP 协议，AI 编程助手可以直接调用业务知识库
- **配置示例（3行）：**
  ```json
  {
    "mcpServers": {
      "omaha": { "command": "python", "args": ["-m", "app.mcp.server"] }
    }
  }
  ```
- **效果：** Claude Code 等工具打开项目后，自动获得查询业务数据的能力
- **意义：** 本体论 + MCP = AI 工具链的标准接入点

---

### Slide 12 — 总结
- **三个带走的观点：**
  1. **本体论** = 给 AI 的业务语义地图，让 AI 真正"理解"领域概念
  2. **配置驱动** = 低门槛扩展，任何领域都可以用 YAML 定义
  3. **MCP** = AI 工具链的标准接口，知识库可被任何兼容工具调用
