### ❌ 测试用例 2.1：按行业筛选股票

**测试目标：** 验证按行业过滤功能

**执行结果：** FAIL

**问题描述：**
- 查询 `industry="银行"` 时，返回的股票不全是银行股
- Tushare API 的 `stock_basic` 接口不支持按 `industry` 字段过滤
- 需要在客户端进行后过滤

**返回数据：**
```
银行股列表:
  1. 000001.SZ - 平安银行
  2. 000002.SZ - 万科Ａ          ← 不是银行股
  3. 000004.SZ - *ST国华         ← 不是银行股
  ...
```

**根本原因：**
Tushare Pro API 的 `stock_basic` 接口只支持以下过滤参数：
- `ts_code`: 股票代码
- `list_status`: 上市状态
- `exchange`: 交易所

不支持 `industry`、`area` 等字段的过滤。

**解决方案：**
1. **方案 A（推荐）：** 在 `_query_tushare()` 方法中添加客户端过滤逻辑
2. **方案 B：** 在文档中明确说明 Tushare API 的限制
3. **方案 C：** 使用 Tushare 的其他 API（如 `ths_index` 获取行业成分股）

## 发现的问题

### 问题 1：Tushare API 过滤限制 ⚠️

**严重程度：** 中等

**描述：**
Tushare Pro API 的 `stock_basic` 接口不支持按 `industry`、`area` 等字段过滤，导致客户端过滤失效。

**影响范围：**
- 按行业筛选股票
- 按地区筛选股票
- 其他非支持字段的过滤

**建议修复：**
在 `_query_tushare()` 方法中添加客户端过滤逻辑：
```python
# 如果 API 不支持某些过滤参数，在客户端进行过滤
unsupported_filters = []
for f in filters:
    if f['field'] not in supported_api_params:
        unsupported_filters.append(f)

# 查询后在客户端过滤
if unsupported_filters:
    for f in unsupported_filters:
        df = df[df[f['field']] == f['value']]
```
