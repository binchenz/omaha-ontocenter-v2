# 轻量级本体（Lightweight Ontology）市场格局分析

> 更新时间：2026-03-17

## 核心概念

"轻量级本体（Lightweight Ontology）"在当下的数据架构中，往往以**"语义层（Semantic Layer）"或"智能元数据图谱（Smart Metadata Graph）"**的形态出现。

### 与重型本体的对比

**Palantir 式重型本体：**
- 需要庞大交付团队梳理所有业务实体
- 构建底层大一统的本体系统
- 重构底层数仓，搬运数据

**轻量级本体：**
- 不搬运数据，不重构底层数仓
- 通过代码（如 YAML）或 AI 自动化
- 在物理数据和业务逻辑之间盖一层"极薄的翻译网"

### 市场趋势

正处于从"古典 BI 数据栈"向"AI Native 数据栈"演进的风口浪尖。

---

## 三大阵营

### 阵营一：现代数据栈的"语义层"基建狂魔

这类企业不直接做最终的图表，他们只做一件事：用极其轻量级的代码定义"业务指标（也就是轻量级本体）"，统一所有数据消费端的口径。

**代表企业：**

1. **dbt Labs (dbt Semantic Layer)**
   - 通过 YAML 定义 metrics（指标）
   - 统一口径，避免"十个人写十个 SQL"
   - 与 dbt 数据转换工具深度集成

2. **Cube.dev**
   - 开源的语义层框架
   - 通过 JavaScript/YAML 定义数据模型
   - 支持多种 BI 工具接入

3. **Looker (LookML)**
   - Google 旗下，通过 LookML 语言定义语义层
   - 强调"单一事实来源"（Single Source of Truth）

**核心价值：**
- 统一指标定义
- 避免重复造轮子
- 降低数据消费门槛

---

### 阵营二：AI Native 的自动化元数据图谱

这类企业的技术含量更高，完美契合"自动化轻量级本体"的概念。

**核心特点：**
- 不依赖用户手动写 YAML
- 通过 AI 自动扫描企业的数据库结构、历史 SQL 查询日志
- 自动在后台构建一个"元数据知识图谱"
- 推断出哪些表经常一起 Join，哪个字段代表特定业务含义
- Agent 在生成查询时，依赖这个动态生成的轻量级图谱作为上下文

**代表企业：**

1. **Secoda**
   - AI 驱动的数据发现和文档平台
   - 自动扫描数据库结构和历史查询
   - 构建动态元数据知识图谱

2. **Dot (GetDot.ai)**
   - 定位为数据助手
   - 强调学习企业的"内部数据方言"
   - 通过轻量级配置文件理解业务上下文
   - 例如："大客户"在当前公司特指"ARR大于10万美金的客户"

**核心价值：**
- 解决"上下文对齐"问题
- 自动化元数据管理
- 降低配置成本

---

### 阵营三：国内市场的演进与空缺

由于企业的数据基建成熟度参差不齐，纯粹的"语义层 SaaS"很难独立存活，往往需要和 BI 结合。

**代表企业：**

1. **Kyligence (跬智信息)**
   - 以前做重型 OLAP 引擎起家
   - 现在推出了 Kyligence Zen（指标平台）
   - 产出的不仅是数据，而是包含业务视角的洞察报告

**市场特点：**
- 数据基建成熟度参差不齐
- 需要语义层 + BI 结合的解决方案
- 强调业务洞察而非纯技术基建

## 关键技术点

### 1. 轻量级定义方式
- YAML/JavaScript 配置文件
- 声明式语法
- 易于版本控制和协作

### 2. AI 自动化
- 扫描数据库结构（Schema）
- 分析历史 SQL 查询日志
- 推断表之间的关系（JOIN 模式）
- 识别字段的业务含义

### 3. 上下文对齐
- 解决业务术语的企业特定含义
- 例如："大客户"、"活跃用户"的定义
- 建立业务语言到数据语言的映射

### 4. 动态元数据图谱
- 不是静态的数据字典
- 随着使用不断学习和优化
- 支持 Agent 智能查询

---

## 投资价值与叙事

这种既懂底层数据架构（大厂数据工程师的硬核基建能力），又具备前沿 Agent 编排理念（AI 开发者的创新思维）的解决方案，在向投资人展示时，是一套逻辑非常自洽且极具爆发力的叙事。

**核心优势：**
1. **技术门槛高**：需要同时掌握数据工程和 AI 技术
2. **市场需求强**：从古典 BI 到 AI Native 的必经之路
3. **可扩展性强**：从语义层基建到 AI Agent 的完整链路
4. **差异化明显**：轻量级 vs 重型本体的清晰定位
